## 2. 技术说明
- **前端框架**：原生 TypeScript（无需React/Vue，因为是纯3D画布项目）
- **3D引擎**：Three.js（three + @types/three）
- **构建工具**：Vite（支持HMR热更新）
- **语言目标**：ES2020
- **模块系统**：ESNext

## 3. 路由定义

## 4. 核心模块说明

### 4.1 文件结构
```
├── package.json          # 项目依赖与脚本
├── index.html            # 入口HTML，全屏canvas容器
├── tsconfig.json         # TypeScript配置（严格模式，ES2020）
├── vite.config.js        # Vite构建配置
└── src/
    ├── main.ts           # 场景初始化、相机、渲染器、事件分发
    ├── particleSystem.ts # 曼陀罗粒子系统（2500光点）
    └── rippleEffect.ts   # 涟漪效果与微型粒子
```

### 4.2 main.ts 职责
- 创建 WebGLRenderer、Scene、PerspectiveCamera
- 初始化 ParticleSystem 和 RippleEffect
- 设置全屏自适应（resize事件）
- 实现 requestAnimationFrame 动画循环
- 处理鼠标事件：
  - mousedown/mousemove/mouseup：拖拽旋转视角（四元数插值）
  - wheel：滚轮缩放
  - click：Raycaster检测点击粒子，触发涟漪
- HUD更新：粒子总数、FPS、视角角度
- UI闲置检测：3秒无操作渐隐HUD

### 4.3 particleSystem.ts 职责
- 创建2500个光点：
  - 分布在扁平圆环：外径6，内径1.5，围绕Z轴对称
  - 颜色按半径渐变：外层暖色(#FF6B6B→#FFD93D)，内层冷色(#6BCB77→#4D96FF→#9B59B6)
  - 尺寸0.08-0.25单位，透明度0.2-0.8
- 使用 THREE.BufferGeometry + THREE.Points + ShaderMaterial 实现高性能渲染
- update(deltaTime) 方法：
  - 整体绕Y轴自转（角速度 2π/20 rad/s）
  - 每个粒子呼吸动画（sin波，1.5-3秒随机周期）
  - 螺旋漂移（偏移0.3单位，8秒周期）

### 4.4 rippleEffect.ts 职责
- 内部涟漪状态机：数组管理多个活跃涟漪
- trigger(worldPosition)：
  - 新建涟漪：半径0→3，1.5秒生命周期
  - 发射10个微型粒子：继承主色，向外飞散，寿命1秒
- update(deltaTime, particleSystem)：
  - 扩散所有活跃涟漪
  - 检测涟漪经过的邻近光点，增大尺寸并闪白0.3秒
  - 更新微型粒子位置与透明度
  - 清理过期涟漪与粒子
- 使用独立的 THREE.Points 渲染微型粒子

## 5. 关键技术实现

### 5.1 相机旋转（四元数平滑插值）
```
- 维护 targetQuaternion 和 currentQuaternion
- 鼠标拖拽时根据偏移量更新 targetQuaternion（球坐标→四元数）
- 每帧使用 Quaternion.slerp 平滑插值
```

### 5.2 粒子系统优化
```
- BufferGeometry存储所有粒子属性（position, color, baseSize, phase等）
- ShaderMaterial在GPU端计算呼吸与漂移，减少CPU负担
- Additive Blending实现半透明叠加辉光
```

### 5.3 点击检测
```
- THREE.Raycaster从鼠标位置发射射线
- 与Points求交，获取最近粒子的索引与世界坐标
```

### 5.4 性能约束
```
- 每帧计算不超过500个涟漪影响粒子
- 微粒子池上限，避免频繁GC
- HUD使用DOM元素，非Canvas渲染
```
