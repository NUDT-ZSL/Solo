### 3.1 物理引擎模块 (physicsEngine.ts)

**Ball类属性**：
```typescript
interface Ball {

  x: number;
  y: number;
  vx: number;
  vy: number;
  radius: number;
  color: string;
  trail: { x: number; y: number }[];
  scale: number; // 弹性缩放动画用
  selected: boolean;
  highlightPhase: number; // 选中闪烁相位
}
```

**Particle类属性**：
```typescript
interface Particle {
  x: number;
  y: number;
  vx: number;
  vy: number;
  radius: number;
  color: string;
  life: number; // 0-1 衰减系数，1=刚生成，0=消失
  maxLife: number; // 总生命周期帧数（约0.3秒=18帧@60FPS）
}
```

**粒子爆散生成算法**：
```
碰撞时生成 8-12 个粒子：
1. 取两球中颜色较亮者作为粒子颜色
   - 亮度计算：Y = 0.299*R + 0.587*G + 0.114*B
2. 粒子半径：2-4px 随机
3. 速度方向：沿碰撞法线两侧 ±60° 范围内随机扩散
   - 法线方向 n = (ballB.x - ballA.x, ballB.y - ballA.y) 归一化
   - 切向方向 t = (-n.y, n.x)
   - 粒子速度 = 随机速度大小 * (n * cos(angle) + t * sin(angle))
   - angle ∈ [-60°, +60°] 随机
4. 速度大小：2-6 px/帧 随机
5. 生命周期：约0.3秒（18帧），线性衰减
6. 生成位置：碰撞接触点
```

**核心函数**：
- `checkCollisions(balls: Ball[], restitution: number, useSpatialGrid: boolean)`: 检测球-球、球-边界碰撞，按动量守恒更新速度
  - 支持网格空间分区（Spatial Grid）优化O(n²)检测
  - 多球碰撞采用迭代求解（多次迭代直至稳定或达到最大迭代次数）
- `resolveBallCollision(ballA: Ball, ballB: Ball, restitution: number)`: 两球碰撞响应，根据质量（半径平方比）计算速度分量
- `spatialGridQuery(balls: Ball[], cellSize: number)`: 空间网格划分，返回潜在碰撞对
- `updatePhysics(balls: Ball[], particles: Particle[], gravity: number, friction: number, canvasWidth: number, canvasHeight: number, frameCount: number)`: 每帧更新位置、应用重力摩擦力、更新拖尾、更新粒子

**碰撞响应详细算法**：
```
1. 计算碰撞法线 n = (x2-x1, y2-y1) / distance
2. 计算相对速度 rv = v2 - v1
3. 相对速度在法线上的投影 rvDotN = rv · n
4. 若 rvDotN > 0，说明两球正在分离，跳过
5. 质量 m1 = π * r1², m2 = π * r2²
6. 冲量标量 j = -(1 + e) * rvDotN / (1/m1 + 1/m2)
   其中 e = 0.85（恢复系数）
7. 冲量向量 impulse = j * n
8. 更新速度：v1 -= impulse/m1, v2 += impulse/m2
9. 位置修正：防止重叠，沿法线方向分离
```

**多球碰撞顺序处理**：
- 采用位置+速度迭代法：每帧多次执行碰撞检测与响应
- 迭代次数：默认3次，球体>200时降为2次
- 每次迭代重新计算所有碰撞对，逐步逼近稳定状态
- 迭代顺序随机化，避免固定顺序导致的偏差

### 3.2 渲染控制模块 (renderer.ts)

**核心函数**：
- `initCanvas(canvas: HTMLCanvasElement)`: 设置Canvas尺寸，处理devicePixelRatio
- `renderScene(ctx: CanvasRenderingContext2D, balls: Ball[], particles: Particle[], selectedId: string | null, isPaused: boolean, canvasWidth: number, canvasHeight: number)`: 绘制整个场景
- `drawBackground(ctx: CanvasRenderingContext2D, w: number, h: number)`: 径向渐变背景
- `drawBallTrail(ctx: CanvasRenderingContext2D, ball: Ball)`: 绘制拖尾轨迹（二次贝塞尔曲线连接最近5帧位置，透明度从0.5线性渐变到0）

**拖尾轨迹绘制算法**：
```
1. 获取trail数组中最近5帧位置点 p0, p1, p2, p3, p4
2. 使用二次贝塞尔曲线平滑连接各点
   - 控制点取相邻两点中点
   - 曲线段：p0→p1 (控制点m01), p1→p2 (控制点m12), 以此类推
3. 透明度从尾部(最早帧)的0线性渐变到头部(最近帧)的0.5
4. 线宽随位置渐变：尾部细，头部粗，与球体半径成比例
```
- `drawBall(ctx: CanvasRenderingContext2D, ball: Ball, isSelected: boolean)`: 绘制球体（含选中高亮）
- `drawParticles(ctx: CanvasRenderingContext2D, particles: Particle[])`: 绘制粒子
- `drawPauseOverlay(ctx: CanvasRenderingContext2D, w: number, h: number)`: 暂停覆盖层

### 3.3 UI控制模块 (uiController.ts)

**键盘方向键微调算法**：
```
方向键控制选中球体的速度向量旋转：
1. 记录当前按下的方向键集合（支持多键同时按下）
2. 根据按下的方向键计算目标旋转方向
   - 单独按上/下/左/右：对应约3度旋转（对应速度变化0.5px/帧）
   - 多键组合（如上+左）：合成方向角度旋转
3. 速度向量旋转公式：
   vx' = vx * cos(θ) - vy * sin(θ)
   vy' = vx * sin(θ) + vy * cos(θ)
   其中 θ = 旋转角度（弧度）
4. 每按一次方向键执行一次旋转（脉冲式，非持续）
5. Delete键删除选中球体（带收缩动画，scale从1→0，时长0.2秒）
```

**UI回调节流机制**：
```
物理参数更新（重力/摩擦力滑动条）使用 requestAnimationFrame 节流：
1. 滑动条 input 事件触发时，将新值存入 pendingValue
2. 使用 rafId 标记是否有待更新的帧
3. 若 rafId 为空，调用 requestAnimationFrame 安排更新
4. 在下一帧中将 pendingValue 应用到物理引擎参数
5. 避免在渲染循环中直接修改 React state 导致竞态条件
```

**回调接口**：
```typescript
interface UICallbacks {
  onGravityChange: (value: number) => void;
  onFrictionChange: (value: number) => void;
  onReset: () => void;

  onKeyUp: (key: string, selectedBallId: string | null) => void;
  onBallCreate: (x: number, y: number, radius: number, color: string) => void;
  onBallSelect: (ballId: string | null) => void;
  onTogglePause: () => void;

```

**核心函数**：
- `createControlPanel(container: HTMLElement, callbacks: UICallbacks)`: 创建控制面板DOM
- `bindCanvasEvents(canvas: HTMLCanvasElement, callbacks: UICallbacks)`: 绑定画布鼠标事件
- `bindKeyboardEvents(callbacks: UICallbacks)`: 绑定键盘事件
- `showColorPicker(x: number, y: number, onSelect: (color: string) => void)`: 显示颜色选择器

## 5. 性能优化策略

1. **碰撞检测优化**：
   - 球体数量 ≤ 200：每帧检测，使用空间网格分区
   - 球体数量 > 200：每2帧检测一次 + 空间网格分区 + 减少碰撞迭代次数(3→2)
   - 空间网格划分（Spatial Grid）：
     ```
     1. 根据最大球体半径确定网格单元大小 cellSize = 2 * maxRadius
     2. 将每个球根据中心坐标放入对应网格单元
        gridKey = (floor(x/cellSize), floor(y/cellSize))
     3. 每个球只需与自身所在单元及相邻8个单元内的球进行碰撞检测
     4. 时间复杂度从 O(n²) 降低到接近 O(n)
     ```
   - 宽相检测：先做距离平方判断，再做精确碰撞响应
   - 使用距离平方比较，避免开方运算

2. **渲染优化**：
   - 拖尾只保留最近5帧
   - Canvas使用devicePixelRatio适配清晰度
   - 离屏缓存静态背景

3. **内存优化**：
   - 粒子对象池复用
   - 及时清除已消失的粒子
   - 避免React频繁重渲染（使用useRef存储高频更新数据）

4. **动画优化**：
   - 全部使用requestAnimationFrame
   - CSS transitions处理UI元素动画
   - 避免布局抖动

## 7. 物理公式

**动量守恒碰撞响应**（一维弹性碰撞扩展到二维）：
```
沿碰撞法线方向分解速度
v1' = ((m1 - e*m2)*v1 + (1+e)*m2*v2) / (m1 + m2)
v2' = ((m2 - e*m1)*v2 + (1+e)*m1*v1) / (m1 + m2)
其中：
  m = π * r²（质量与面积成正比）
  e = 0.85（恢复系数）
```

**动能计算**：
```
KE = Σ 0.5 * m * (vx² + vy²)
单位：J（焦耳），质量单位取相对值
```
