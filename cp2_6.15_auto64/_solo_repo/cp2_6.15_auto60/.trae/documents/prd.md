## 1. 产品概述

基于用户心率和步频数据实时生成个性化运动歌单的 Web 应用，让运动爱好者不再为跑步或骑行时选歌发愁。系统根据智能手表同步的实时心率（模拟数据）和运动速度，自动匹配 BPM 合适的歌曲，并动态调整节奏，让音乐与运动节奏同步提升沉浸感。

- 目标用户：跑步、骑行等运动爱好者
- 核心价值：音乐节奏与运动节奏实时匹配，提升运动体验

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 使用模拟运动功能，浏览歌单，查看运动统计 |

### 2.2 功能模块
1. **运动状态面板**：实时心率、步频展示，数值变化动画
2. **动态歌单列表**：按 BPM 匹配度排序，卡片入场动画，点击进入详情
3. **歌曲详情页**：波形图、频谱动画，歌曲详细信息
4. **运动统计侧边栏**：总运动时长、平均心率、卡路里估算、心率曲线图
5. **模拟运动控制**：开始/停止模拟运动，数据自然波动

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页面 | 运动状态卡片 | 展示当前心率、步频，心率数值颜色随强度变化 |
| 主页面 | 动态歌单列表 | 按 BPM 匹配度排序，卡片滑入动画，匹配度高的歌曲发光 |
| 主页面 | 底部控制按钮 | 开始/停止模拟运动，渐变按钮悬停缩放效果 |
| 主页面 | 侧边统计栏 | 运动时长、平均心率、卡路里、迷你心率曲线图 |
| 详情页 | 波形图展示 | Canvas 绘制波形，频谱条随节奏跳动 |
| 详情页 | 歌曲信息 | 歌曲名称、艺术家、BPM 等详细信息 |

## 3. 核心流程

用户点击"开始模拟运动"按钮 → 系统启动模拟数据流（每2秒更新心率和步频）→ 心率数据自然波动（先升后稳再略降）→ Web Worker 计算 BPM 匹配度 → 歌单自动重新排序（AnimatePresence 过渡动画）→ BPM 匹配度最高的三首歌置顶并发光 → 用户点击歌曲卡片 → 右滑过渡进入详情页 → 展示波形图和频谱动画 → 用户点击返回 → 左滑过渡回到歌单列表

## 4. 用户界面设计

### 4.1 设计风格
- **主题**：深色运动主题，科技感与活力并存
- **主背景**：#121212（深黑）
- **卡片背景**：#1e1e1e（深灰）
- **点缀色**：紫色/青色/粉色渐变
- **圆角**：卡片 16px，歌曲卡片 12px，按钮 24px
- **字体**：数值使用 monospace 字体，增强数据感

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 主页面 | 运动状态卡片 | 宽280px高120px，心率数值颜色动态变化（绿/橙/红），数字脉冲动画 |
| 主页面 | 歌单列表 | 宽60%，水平卡片高70px，BPM匹配度渐变背景，左滑入场动画 |
| 主页面 | 控制按钮 | 宽200px，粉紫渐变，悬停缩放1.05倍 |
| 主页面 | 侧边栏 | 宽200px，背景#1a1a1a，统计数值计数动画，迷你折线图 |
| 详情页 | 波形图 | Canvas绘制，频谱条8-48px高度，青到紫渐变 |

### 4.3 响应式
- **桌面端**：三栏布局（侧边栏 + 歌单 + 状态卡片）
- **移动端**（<768px）：侧边栏隐藏为汉堡菜单，歌单宽度100%，字体间距自适应缩小
- **汉堡菜单**：三横线变X动画，过渡0.3秒

### 4.4 动效设计
- 心率数字脉冲动画：0.3秒缩放
- 歌曲卡片入场：左边缘滑入，0.5秒 cubic-bezier
- 页面过渡：右滑/左滑 0.4秒
- 歌单排序：AnimatePresence 过渡 0.6秒
- Top3发光边框：脉冲动画 1.5秒 infinite
- 数值变化：计数动画 1秒
- 按钮悬停：缩放 1.05倍
