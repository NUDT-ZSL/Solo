## 2. 技术描述

- **前端框架**：React 18 + TypeScript 5（严格模式）
- **构建工具**：Vite 5 + @vitejs/plugin-react
- **渲染引擎**：Canvas 2D API + 双缓冲离屏Canvas
- **物理模拟**：简化Navier-Stokes方程（Jos Stam稳定流体求解器）
- **状态管理**：React Hooks (useState / useRef / useEffect)
- **样式方案**：原生CSS + CSS自定义属性（深色主题变量）
- **无后端依赖**，纯前端单页应用

**调用关系与数据流：**
1. `App.tsx` 通过 `useRef<HTMLCanvasElement>` 将画布DOM传入 `interactions.ts` 初始化
2. `interactions.ts` 在每帧前通过闭包变量累积注入事件数组，`App.tsx` 每帧读取并清空
3. `App.tsx` 在RAF回调中调用 `fluidEngine.step(timestamp, injections, params)` → 返回 `boolean`（渲染完成信号）
4. `controls.tsx` 接收当前参数值和 `onChange` 回调，所有变更同步到 `App.tsx` state
5. `fluidEngine.ts` 内部维护 4 个 Float32Array 场（速度X/Y 双缓冲、密度RGB 双缓冲），无外部状态依赖

## 4. 核心类型定义

```typescript
// 单条注入数据（interactions.ts → App.tsx → fluidEngine.ts）
export interface Injection {
  x: number;          // 画布归一化X坐标 (0~1)
  y: number;          // 画布归一化Y坐标 (0~1)
  dx: number;         // 速度向量X（像素/帧）
  dy: number;         // 速度向量Y（像素/帧）
  color: [number, number, number];  // RGB 0~255
  radius: number;     // 注入半径（0~1，相对画布短边）
}

// 流体引擎参数（App.tsx → fluidEngine.ts）
export interface FluidParams {
  viscosity: number;          // 粘度 1~20，越大扩散越慢
  dissipation: number;        // 颜色衰减率 0.95~0.99
  resolutionScale: number;    // 渲染分辨率比例 0.25~1.0
  colorMode: 'single' | 'gradient' | 'rainbow';
  injectColor: string;        // HEX色值，单一模式下使用
  gradientA: string;          // 渐变起始色
  gradientB: string;          // 渐变结束色
  gradientPeriod: number;     // 渐变周期(ms)，默认5000
  rainbowPeriod: number;      // 彩虹周期(ms)，默认10000
}

// 控制面板向外暴露的变更回调（controls.tsx → App.tsx）
export interface ControlCallbacks {

  onDissipationChange: (v: number) => void;
  onResolutionChange: (v: number) => void;
  onColorModeChange: (mode: FluidParams['colorMode']) => void;
  onInjectColorChange: (hex: string) => void;
  onBackgroundPreset: (presetIndex: number) => void;
  onReset: () => void;
  onAutoPaintToggle: (enabled: boolean) => void;
}
```

## 5. 流体引擎核心算法说明

基于 Jos Stam 2003年《Stable Fluids》的简化实现，每帧循环如下：

1. **外力注入** (addForce)：将鼠标/自动注入的速度和RGB密度加到对应网格
2. **粘性扩散** (diffuse)：速度场扩散，迭代次数与viscosity正相关，高斯-赛德尔求解
3. **投影步骤1** (project divergence)：计算散度场
4. **投影步骤2** (project pressure)：求解压力泊松方程
5. **投影步骤3** (project subtract)：减去压力梯度以保证不可压缩性
6. **速度平流** (advect velocity)：通过半拉格朗日法沿速度场回溯自身
7. **密度平流** (advect density)：颜色场沿速度场运动，使用双线性插值
8. **衰减** (dissipate)：密度场乘以 dissipation 系数
9. **渲染**：将RGB密度场从低分辨率Float32Array通过ImageData绘制到离屏Canvas，再缩放拷贝到显示Canvas

## 6. 性能预算

| 指标 | 要求 |
|------|------|
| 默认分辨率(1.0x) FPS | ≥ 55 |
| 半分辨率(0.5x) FPS | ≥ 58 |
| 双缓冲Canvas内存 | ≤ 200MB（含速度/密度Float32Array） |
| 模拟网格分辨率 | 1.0x下约 256×256 ~ 512×512（视口尺寸自动调整） |
| 单次step耗时（JS主线程） | ≤ 12ms |
