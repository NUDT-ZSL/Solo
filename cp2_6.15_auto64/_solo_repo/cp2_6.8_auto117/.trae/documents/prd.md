## 1. 产品概述

交互式排序算法可视化对比工具，面向编程入门学生，通过并排动画展示冒泡、选择、快速三种排序算法的执行过程，帮助理解算法原理与时间复杂度差异。

- 主要目的：让学生直观观察算法每一步的比较与交换操作，通过动画对比不同算法的效率
- 目标用户：编程入门课程学生、算法学习者
- 产品价值：将抽象的排序算法过程可视化、可交互、可对比，提升学习效率和理解深度

## 2. 核心功能

### 2.1 功能模块

1. **数组输入模块**：支持用户输入逗号分隔整数数组，或一键生成随机数组（6-15个元素，范围10-100）
2. **算法执行控制**：点击"开始排序"后同步启动三个算法的动画演示
3. **冒泡排序可视化窗口**：显示冒泡排序动画、比较次数、交换次数、进度百分比
4. **选择排序可视化窗口**：显示选择排序动画、比较次数、交换次数、进度百分比
5. **快速排序可视化窗口**：显示快速排序动画、比较次数、交换次数、进度百分比

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主页面 | 顶部标题栏 | 显示应用名称与简介 |
| 主页面 | 数组输入区 | 文本输入框 + 生成随机数组按钮 + 开始排序按钮 |
| 主页面 | 排序算法展示区 | 三个并排的排序可视化窗口（冒泡/选择/快速） |
| 主页面 | 单排序窗口 | 条柱动画区 + 算法信息（名称/比较/交换/进度）+ 完成提示 |

## 3. 核心流程

用户在输入框输入数组或生成随机数组 → 点击"开始排序"按钮 → 三个排序算法窗口同步逐帧推进动画 → 每步高亮比较/交换的条柱 → 完成后展示渐变彩色条柱与完成提示。

```mermaid

    D --> E["显示比较次数/交换次数/进度"]
    E --> F["排序完成，展示完成态"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深灰背景 #2D3748，浅灰标题栏 #EDF2F7，窗口背景 #E2E8F0
- **辅助色**：比较高亮 #F6E05E（黄），交换高亮 #ED8936（橙），基础蓝色 #4299E1，完成渐变 #48BB78→#9F7AEA（绿→紫）
- **按钮样式**：蓝色 #4299E1 背景、白色文字、圆角 6px，悬停变暗为 #3182CE
- **字体**：使用现代无衬线字体，标题较大醒目，正文清晰易读
- **布局**：顶部标题栏 → 输入区 → 水平 flex 三栏窗口展示
- **动画**：所有过渡使用 ease-in-out 缓动函数，时长统一 0.3 秒

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|---------|---------|---------|
| 主页面 | 顶部标题栏 | 浅灰背景，居中标题文字 |
| 主页面 | 数组输入区 | 文本输入框、两个蓝色按钮，水平排列 |
| 主页面 | 排序展示容器 | 水平 flex，窗口间距 16px，深灰背景 |
| 主页面 | 排序窗口卡片 | 浅灰背景 #E2E8F0、圆角 8px、内边距 12px、阴影 0 2px 6px rgba(0,0,0,0.2) |
| 主页面 | 条柱容器 | 窗口宽度 320px，条柱高度映射数值，宽度自适应 |
| 主页面 | 条柱动画 | CSS transition 平滑平移，颜色渐变切换 |
| 主页面 | 完成态 | 条柱绿紫渐变 + 顶部金色光晕闪烁 + 绿色完成文字渐显 |

### 4.3 响应式设计

- 桌面端优先：窗口宽度 ≥ 900px 时三个窗口水平并排，每窗口宽 320px
- 移动端适配：窗口宽度 < 900px 时改为垂直堆叠，每个窗口宽度 100%
- 触摸优化：按钮足够大便于点击

### 4.4 性能约束

- 动画帧率：60fps
- 15 个元素总动画时长 ≤ 15 秒（每步约 0.1 秒）
- 内存占用 ≤ 50MB
