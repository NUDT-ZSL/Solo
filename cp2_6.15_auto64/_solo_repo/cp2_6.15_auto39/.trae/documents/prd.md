## 1. 产品概述

虚拟厨房协同菜谱管理应用，旨在为美食博主和家庭烹饪爱好者提供一站式菜谱管理、食材用量调整和协同购物清单解决方案。

- 核心问题：用户在多个菜谱网站间切换，无法统一管理收藏、缩放食材用量、生成购物清单，多人聚餐时菜品协调困难
- 目标用户：美食博主、家庭烹饪爱好者、多人聚餐组织者
- 产品价值：集中管理菜谱收藏，智能调整食材用量，实时协同生成购物清单

## 2. 核心功能

### 2.1 功能模块

1. **菜谱管理模块**：菜谱浏览搜索、菜谱详情展示、食材用量缩放、收藏与评分
2. **协同购物清单模块**：食材汇总分类、单位统一换算、购物清单生成、实时协同编辑

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 菜谱列表页 | 搜索过滤 | 按菜名和食材关键词实时搜索，300ms内响应 |
| 菜谱列表页 | 菜谱卡片网格 | 每行3张卡片（平板2张），展示缩略图、菜名、作者、评分、收藏 |
| 菜谱详情页 | 步骤说明 | 左侧展示带编号的烹饪步骤，浅灰分割线分隔 |
| 菜谱详情页 | 食材列表 | 右侧展示食材名称、原始用量、可编辑调整系数滑块（1%-500%） |
| 购物清单页 | 食材分组 | 按类别（蔬菜/肉类/调料等）分组展示，带柔和底色标签 |
| 购物清单页 | 数量与价格 | 合计数量（统一单位）、可编辑单价、实时计算总价 |
| 购物清单页 | 实时协同 | WebSocket同步编辑，他人修改闪烁提示和虚线框标记 |

## 3. 核心流程

用户浏览菜谱列表 → 搜索/筛选感兴趣的菜谱 → 点击卡片查看详情 → 调整食材用量系数 → 勾选多个菜谱 → 生成购物清单 → 通过分享链接邀请他人 → 多人协同编辑清单（实时同步）

```mermaid
    D --> E["勾选多个菜谱"]

    G --> H["多人实时编辑清单"]
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：浅绿 `#a8e6cf`，悬停加深至 `#7dcbb5`
- 背景色：米白 `#fef9ef`
- 卡片背景：白色到浅灰 `#fafafa` 渐变
- 滑块轨道渐变：蓝色 `#4fc3f7` 到绿色 `#81c784`
- 圆角：卡片16px，分类标签6px
- 卡片尺寸：280px × 320px，缩略图占高度40%
- 布局：居中1200px容器，卡片/区块间距24px

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 菜谱列表页 | 搜索栏 | 浅绿色调输入框，实时搜索 |
| 菜谱列表页 | 菜谱卡片 | 渐变背景、圆角、缩略图、星级评分、收藏心形动画 |
| 菜谱详情页 | 步骤面板 | 编号列表、浅灰分割线 |
| 菜谱详情页 | 食材面板 | 滑块控件、实时数值高亮、渐变轨道 |
| 购物清单页 | 分类分组 | 柔和底色标签、圆角6px |
| 购物清单页 | 清单项 | 勾选框、可编辑数量/单价、总价实时更新 |
| 购物清单页 | 协同提示 | 黄色高亮渐隐、虚线框标记 |

### 4.3 响应式

- 桌面（>1200px）：卡片每行3张，标准字号
- 平板（>768px ≤1200px）：卡片每行2张，标题字号缩小
- 采用桌面优先设计，媒体查询适配平板
