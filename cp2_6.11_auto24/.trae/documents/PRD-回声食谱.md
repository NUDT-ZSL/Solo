## 1. 产品概述

「回声食谱」是一款为热爱烹饪但常记不住做菜顺序的人群设计的交互式菜谱导航工具。用户输入菜谱文本后，系统自动解析步骤并生成带计时器的可交互流程图，帮助用户有序完成烹饪流程。

- 核心目标：降低烹饪过程中的步骤记忆负担，通过可视化流程图 + 智能计时 + 自动引导
- 目标用户：家庭主妇、烹饪爱好者、厨房新手
- 市场价值：填补菜谱从"阅读菜谱"到"按流程操作"之间的断层，提升烹饪体验

## 2. 核心特性

### 2.1 功能模块

1. **菜谱输入页**：文本输入区域、菜谱列表、示例菜谱快速加载
2. **流程图导航页**：步骤卡片流程图、计时联动、自动滚动、连接线光效

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 菜谱输入页 | 文本输入区 | 支持粘贴/输入菜谱文本，格式为"步骤X: 操作文本 时长" |
| 菜谱输入页 | 解析预览 | 实时预览解析后的步骤卡片 |
| 菜谱输入页 | 菜谱列表 | 展示已保存的菜谱，支持CRUD |
| 流程图导航页 | 步骤卡片 | 展示食材、操作文字、倒计时进度条、配图 |
| 流程图导航页 | 连接线 | 步骤间带箭头的渐变流动光效连接线 |
| 流程图导航页 | 计时控制 | 播放/暂停按钮、SVG圆形进度条、颜色渐变 |
| 流程图导航页 | 自动滚动 | 当前步骤高亮并自动滚动至视野中央 |

## 3. 核心流程

用户打开应用 → 输入/选择菜谱 → 系统解析步骤 → 点击开始烹饪 → 逐步执行步骤 → 计时完成提醒 → 自动进入下一步 → 全部完成

```mermaid
flowchart LR
    A["进入应用"] --> B["输入菜谱文本"]
    B --> C["系统正则解析"]
    C --> D["渲染流程图"]
    D --> E["点击开始烹饪"]
    E --> F["当前步骤高亮滚动"]
    F --> G["点击播放计时"]
    G --> H["倒计时进行中"]
    H --> I{"计时结束?"}
    I -- 否 --> H
    I -- 是 --> J["弹出成功提示+叮咚声"]
    J --> K{"还有下一步?"]
    K -- 是 --> L["自动切换下一步"]
    L --> F
    K -- 否 --> M["烹饪完成"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：
  - 背景色：米白 `#FFF8F0`
  - 主文字/边框：深棕 `#2D1810`
  - 连接线渐变：`#FF6B35` → `#F7C948`
  - 进度条渐变：红 `#E53E3E` → 绿 `#38A169`

- **卡片样式**：
  - 圆角：`12px`
  - 悬浮阴影：`0 4px 12px rgba(45,24,16,0.15)`
  - Hover效果：阴影加深 + 上浮 `2px`，过渡 `0.3s ease

- **字体**：暖色系字体，标题加粗，正文常规字重适中

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 菜谱输入页 | 顶部标题栏 | 品牌名称，暖色调，居中布局 |
| 菜谱输入页 | 文本输入区 | 大文本框，提示文字，圆角边框 |
| 菜谱输入页 | 解析按钮 | 暖色填充按钮，圆角 |
| 流程图导航页 | 卡片容器 | 横向滚动区域，卡片300px宽度 |
| 流程图导航页 | 步骤卡片 | 白色背景，暖棕边框，圆角12px |
| 流程图导航页 | 进度环 | SVG描边动画，颜色随进度渐变 |
| 流程图导航页 | 连接线 | SVG渐变流动光点，1秒移动一次 |

### 4.3 响应式适配

- **桌面端**（≥1024px）：卡片宽300px，两列布局
- **平板端**（768-1023px）：卡片宽280px，单列布局
- **手机端**（<768px）：卡片宽100%，居中单列，所有元素自适应重排

### 4.4 动画与交互

- 步骤切换：`scroll-behavior: smooth` 平滑滚动
- 卡片展开：点击展开详细说明和配图
- 计时完成：淡入 + 上移0.5秒动画
- 连接线光点：每1秒流动一次
- 所有动画保持60fps流畅体验
