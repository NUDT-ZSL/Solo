## 1. 产品概述

「味觉共振」是一个风味社交平台，用户可以记录品尝食物时的口感、气味和心情，系统基于记录生成动态"风味网络图"，并发现风味相似的食物和其他用户。
- 目标用户：美食爱好者、料理创作者、风味探索者
- 核心价值：将个人的味觉体验可视化、社交化，让食物不再只是"好吃/不好吃"，而是一张可探索的风味地图

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 免注册即可浏览公开档案 | 浏览首页卡片、查看详情页、搜索食物 |
| 注册用户 | 用户名+密码注册 | 创建风味记录、点赞、收藏、关注他人、管理个人风味档案 |

### 2.2 功能模块

1. **首页**：公开风味档案卡片瀑布流、搜索与筛选、风味标签导航
2. **详情页**：风味雷达图、相似食物推荐、评论区、点赞/收藏操作
3. **创建页**：风味记录表单（食物名称、口感维度评分、气味描述、心情emoji、食物图片）
4. **风味网络图**：基于用户数据动态生成的食物节点关系图
5. **个人中心**：我的风味档案、收藏列表、关注列表

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 首页 | 卡片瀑布流 | 展示最近公开风味档案卡片，含食物缩略图、简短描述、风味标签、心情emoji；毛玻璃质感卡片，悬停微上浮+阴影加深 |
| 首页 | 搜索与筛选 | 按食物名称、风味标签、心情类型筛选 |
| 详情页 | 风味雷达图 | 六维口感可视化（甜、咸、酸、苦、鲜、辣） |
| 详情页 | 相似食物推荐 | 基于风味相似度的食物列表，显示相似度百分比 |
| 详情页 | 评论区 | 用户可发表评论，展示评论列表 |
| 详情页 | 互动操作 | 点赞、收藏按钮，带缓动动画反馈 |
| 创建页 | 风味记录表单 | 食物名称输入、六维口感滑块、气味文本描述、心情emoji选择器、图片上传（模拟） |
| 风味网络图 | 动态关系图 | 食物节点+相似度连线，节点大小=喜好度，节点颜色=情感倾向（暖色=喜欢，冷色=不喜欢） |
| 个人中心 | 档案管理 | 查看和编辑自己的风味记录列表 |
| 个人中心 | 收藏与关注 | 收藏的风味档案、关注的用户列表 |

## 3. 核心流程

用户打开首页浏览公开风味档案卡片 → 点击感兴趣的卡片进入详情页 → 查看风味雷达图和相似食物推荐 → 点赞或收藏 → 创建自己的风味记录 → 系统分析并更新风味网络图 → 发现风味相似的食物和用户

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：米白（#FAF6F1）到浅棕（#E8DDD3）渐变背景
- 强调色：暖橙（#D4845A），辅助色：深棕（#6B4C3B）
- 按钮风格：圆角柔和，带微阴影，hover缓动上浮
- 字体：正文用思源宋体/Noto Serif SC，标题用 Playfair Display
- 布局风格：卡片式瀑布流，顶部导航栏
- 图标/emoji：风味标签用圆形色块+文字，心情用emoji表情
- 动画：卡片hover上浮0.3s ease-out，按钮click缩放0.15s，页面切换fade+slide

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 首页 | 卡片瀑布流 | 米白渐变背景、毛玻璃卡片（backdrop-blur+半透明白）、食物缩略图左上、风味标签彩色胶囊、心情emoji右上、悬停上浮8px+阴影加深 |
| 首页 | 顶部导航 | 透明底+模糊背景、logo居左、搜索框居中、创建按钮居右、暖橙色强调 |
| 详情页 | 风味雷达图 | Canvas绘制六边形雷达图、暖橙色填充区域、深棕色轴线、维度标签 |
| 详情页 | 相似食物列表 | 水平滚动卡片、小尺寸缩略图+相似度百分比条 |
| 详情页 | 评论区 | 底部输入框+发送按钮、评论卡片带头像+用户名+时间 |
| 创建页 | 表单 | 滑块组件（六维口感）、emoji选择网格、大号提交按钮 |
| 风味网络图 | 关系图 | Canvas/SVG绘制、力导向布局、节点hover高亮关联连线 |

### 4.3 响应式

- 桌面优先设计，最大宽度1200px居中
- 移动端：卡片单列、导航折叠为汉堡菜单、详情页竖向排列
- 触摸优化：按钮最小44px点击区域、滑块触摸友好

### 4.4 性能要求

- 交互帧率保持60fps
- 卡片列表虚拟滚动（超过50张卡片时）
- 图片懒加载
- CSS动画优先，JS动画使用requestAnimationFrame
