# 豆录 - 咖啡品鉴档案应用 产品需求文档

## 1. 产品概述
豆录是一款面向咖啡爱好者的品鉴记录与社交应用，帮助用户系统化地记录个人咖啡品鉴经验，并通过盲猜挑战与其他爱好者进行趣味互动。
- 核心价值：解决咖啡品鉴经验难以量化记录、风味描述过于主观、爱好者之间缺乏有趣互动方式的痛点
- 目标用户：手冲咖啡爱好者、精品咖啡从业者、咖啡品鉴初学者

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 模拟账号注册 | 记录品鉴日志、参与盲猜挑战、查看个人统计、浏览排行榜 |

### 2.2 功能模块
1. **首页**：品鉴日志瀑布流展示、添加新日志入口
2. **挑战页**：香气风味盲猜挑战、实时排行榜
3. **个人中心**：风味偏好统计仪表盘、个人品鉴记录回顾

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 首页 | 顶部导航 | 应用Logo + 三个页面入口（首页/挑战/个人中心） |
| 首页 | 瀑布流卡片列表 | 按日期倒序展示品鉴记录卡片，卡片包含豆子信息、风味标签、操作按钮 |
| 首页 | 添加品鉴记录弹窗 | 表单录入产地、烘焙度、处理法、风味勾选、冲煮参数、照片上传 |
| 挑战页 | 挑战面板 | 展示两支豆子的风味描述（隐藏产地名称），提供猜测选项 |
| 挑战页 | 排行榜 | 实时显示前10名挑战者排名、头像、得分，前三名特殊颜色标注 |
| 个人中心 | 风味雷达图 | 展示五大类风味偏好比例，支持切换统计周期（30天/90天/全部） |
| 个人中心 | 个人记录回顾 | 展示用户自己的品鉴记录列表 |

## 3. 核心流程
用户打开应用后浏览首页品鉴卡片，可以添加自己的品鉴记录；进入挑战页参与风味盲猜，猜对获得积分并刷新排行榜；在个人中心查看自己的风味偏好统计。

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：大地色系，主色 #8B6F5E（咖啡棕）、辅色 #D4A373（焦糖）、背景 #FAF5EB（奶咖色）、文字 #3E2723（深咖）
- **按钮样式**：主色填充、圆角8px，悬停变暗10%，按压缩小至0.95倍
- **字体**：标题使用 Noto Serif SC（衬线），正文使用 Noto Sans SC（无衬线）
- **布局风格**：卡片式设计，首页瀑布流布局
- **图标风格**：Lucide React 线性图标

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 首页 | 瀑布流卡片 | 280px宽卡片、圆角12px、#FAF5EB背景、2px #D4C5B0阴影，标题#5D4037加粗，悬停上浮8px加深阴影 |
| 挑战页 | 排行榜 | 顶部展示前10名，头像48px圆形，前三名得分分别为#FFD700/#C0C0C0/#CD7F32，数据切换0.4s淡入淡出 |
| 个人中心 | 雷达图 | 浅色#F0EBE1背景、圆角12px，切换周期0.5s形状动画 |
| 全站 | 导航栏 | 固定顶部56px，白色半透明90%背景，底部1px #D4C5B0分割线，flex居中布局 |

### 4.3 响应式适配
- **桌面端（≥1280px）**：瀑布流多列展示，导航栏完整显示三个入口
- **移动端（320-480px）**：瀑布流单列展示，导航栏折叠为汉堡菜单
- 所有过渡动画统一使用 0.3s ease-out

### 4.4 性能要求
- 瀑布流滚动保持 60fps
- 首屏加载 ≤ 2秒（图片懒加载、代码分割）
