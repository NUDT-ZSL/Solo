## 4. API定义

### 类型定义

```typescript
// 匹配状态枚举
type MatchStatus = 'comparing' | 'matched' | 'mismatched' | 'final-matched';

// 单个步骤快照
interface StepSnapshot {
  textIndex: number;        // 当前比较的文本索引
  patternIndex: number;     // 当前比较的模式索引
  matchStatus: MatchStatus; // 匹配状态
  comparisonCount: number;  // 累计比较次数
  patternStart: number;     // 本轮模式在文本中的起始位置
  matchedPositions: number[]; // 已发现的所有匹配起始位置
}

// 算法类型
type AlgorithmType = 'KMP' | 'Boyer-Moore';
```

### AlgorithmEngine 公共接口

```typescript
function generateSnapshots(
  text: string,
  pattern: string,
  algorithm: AlgorithmType
): StepSnapshot[]
```

## 5. 数据模型

不涉及数据库，所有数据均在内存中通过React state管理：

```typescript
// App.tsx 状态模型
interface AppState {
  text: string;                    // 用户输入的文本
  pattern: string;                 // 用户输入的模式
  algorithm: AlgorithmType;        // 选中的算法
  snapshots: StepSnapshot[];       // 步骤快照数组
  currentStep: number;             // 当前步进索引
  isPlaying: boolean;              // 是否自动播放中
  isStarted: boolean;              // 是否已开始匹配
}
```

## 6. 核心算法实现说明

### KMP算法
- 预处理：计算next数组（最长公共前后缀）
- 匹配：失配时利用next数组跳过不必要的比较，模式向右滑动`j - next[j]`位

### Boyer-Moore算法
- 预处理：构建坏字符表
- 匹配：从右向左比较，失配时利用坏字符规则计算滑动距离
