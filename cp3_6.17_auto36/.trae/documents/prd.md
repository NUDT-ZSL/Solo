## 1. 产品概述

共享办公空间设备借用与信用评级应用，旨在为自由职业者提供统一的设备借还登记系统，解决设备找不到主人、归还超时等问题。目标用户为共享办公空间的自由职业者和设备管理员，通过信用评分机制激励按时归还，保障设备流转效率。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 系统预设账户 | 浏览设备、发起借用、查看信用评分、查看借用记录 |
| 管理员 | 系统预设账户 | 管理设备清单、处理归还确认、查看所有借用记录、标记超时归还 |

### 2.2 功能模块

1. **设备总览页**：网格展示所有可借设备卡片，含缩略图、名称、类型标签、状态徽章和借用按钮
2. **设备详情页**：大图展示、技术参数、历史借用记录、借用操作
3. **用户档案页**：头像、信用评分进度条、借用历史表格
4. **管理面板页**：所有借用记录列表、设备管理、归还确认操作
5. **借用流程**：确认模态框 → API调用 → 二维码模态框（供扫码确认）

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 设备总览页 `/overview` | 设备网格 | 以响应式网格（4/3/2列）展示设备卡片，每张卡片显示缩略图、名称、类型标签、状态徽章（绿色空闲/黄色被借/红色维修）及借用按钮 |

## 3. 核心流程

### 3.1 设备借用流程
用户在总览页或详情页点击借用按钮 → 弹出确认模态框 → 确认后调用后端API创建借用记录 → 设备状态变为"被借" → 前端显示二维码模态框（含借用记录ID）→ 管理员扫码确认 → 借用完成

### 3.2 设备归还流程
管理员在管理面板查看未归还记录 → 点击确认归还 → 系统判断是否超时 → 更新用户信用分（按时+1，超时-5）→ 设备状态变为"空闲"

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深蓝 `#1e293b`，辅色：灰白 `#f8fafc`
- 按钮风格：圆角8px，主色深蓝，禁用态 `#94a3b8`，悬浮色变深
- 字体：思源黑体 / Noto Sans SC，标题16-24px，正文14px
- 布局风格：顶部固定导航栏 + 内容区卡片网格布局
- 图标风格：线性图标（Lucide）

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 全局 | 导航栏 | 固定60px高，背景#1e293b，左侧Logo"设备共享站"，右侧导航链接，当前页下划线渐变蓝#3b82f6 |
| 设备总览页 | 设备卡片网格 | 卡片宽240px高320px，圆角12px，白底，阴影0 2px 8px rgba(0,0,0,0.08)，悬浮上移4px阴影加深，间距24px |
| 设备总览页 | 状态徽章 | 绿色空闲#22c55e / 黄色被借#eab308 / 红色维修#ef4444 |
| 设备详情页 | 设备大图 | 宽100%高auto，圆角8px |
| 设备详情页 | 信用分不足提示 | 红色文字提示 |
| 用户档案页 | 头像 | 圆形60px，边框2px solid #e5e7eb |
| 用户档案页 | 信用评分进度条 | 0-100分，颜色从#ef4444渐变到#22c55e |
| 用户档案页 | 借用历史表格 | 状态色块：绿#22c55e / 黄#eab308 / 红#ef4444 |
| 二维码模态框 | 弹出层 | 半透明背景#00000080，居中白底圆角12px，二维码256px，留白16px，右上角关闭按钮 |

### 4.3 响应式设计
- 大屏（≥1024px）：每行4张卡片
- 中屏（768-1023px）：每行3张卡片
- 小屏（<768px）：每行2张卡片，卡片宽100%高度自适应
- 桌面优先设计，移动端适配触摸操作

### 4.4 性能目标
- 首次加载完成时间 ≤ 2秒
- API请求响应时间 ≤ 500毫秒
- 列表分页每页最多20个
- 页面切换保持60fps流畅
- 用户操作反馈延迟 ≤ 100毫秒
