## 1. 产品概述

本产品是面向民间戏剧团体的巡演排期与观众数据分析平台，解决小剧团难以高效管理多个剧目巡演日程、票房统计以及观众偏好画像的痛点。

- 核心价值：为剧团提供一站式巡演管理工具，提升运营效率，数据驱动决策
- 目标用户：剧团管理员、演出策划人员、普通观众

## 2. 核心功能

### 2.1 用户角色

| 角色 | 使用方式 | 核心权限 |
|------|----------|----------|
| 剧团管理员 | 后台管理 | 剧目管理、巡演排期、票房数据查看 |
| 观众 | 前台浏览 | 浏览演出日历、选座购票、查看电子票 |

### 2.2 功能模块

1. **剧目管理模块**：剧目卡片列表展示、搜索筛选、新增编辑表单
2. **巡演地图模块**：全国地图标记演出站点、站点详情弹窗、巡演路线展示
3. **数据分析模块**：票房趋势折线图、时间范围筛选
4. **观众购票模块**：巡演日历浏览、座位选择、模拟购票、电子票生成

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 剧目管理页 | 剧目列表 | 垂直卡片展示、按类型筛选、关键字搜索 |
| 剧目管理页 | 剧目表单 | 新增/编辑剧目信息（名称、类型、时长、演员、海报） |
| 巡演地图页 | 地图展示 | 全国地图、站点标记、曲线箭头路线连接 |
| 巡演地图页 | 站点弹窗 | 演出剧目、时间、票房数据展示 |
| 数据分析页 | 票房图表 | 每日/每周售票量折线图、时间范围筛选 |
| 观众购票页 | 演出日历 | 按城市和日期筛选演出 |
| 观众购票页 | 座位选择 | 座位状态可视化（可选/已售/已占） |
| 观众购票页 | 电子票 | 模拟购票后生成电子票卡片 |

## 3. 核心流程

### 剧团管理员流程
管理员登录 → 进入剧目管理 → 创建/编辑剧目 → 进入巡演地图 → 查看各站点数据 → 进入数据分析 → 查看票房趋势

### 观众购票流程
观众进入首页 → 浏览巡演日历 → 按城市/日期筛选 → 选择演出 → 选择座位 → 确认购票 → 生成电子票

## 4. 用户界面设计

### 4.1 设计风格

- **主题**：深色主题，主背景 #1a1a2e，导航栏 #16213e，卡片 #0f3460
- **文字**：主色 #e0e0e0，强调色 #e94560
- **按钮**：悬浮时有颜色渐变动画（0.2s ease）
- **输入框**：获得焦点时边框变为 #e94560 并带微光效果
- **卡片**：剧目卡片 240×360px，圆角 16px，悬浮抬起 6px
- **字体**：选用现代无衬线字体，保证深色背景下的可读性

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 剧目管理页 | 剧目卡片 | 海报区域、剧目摘要、悬浮抬升效果 |
| 巡演地图页 | 地图标记 | 站点标记、弹窗详情、曲线箭头路线 |
| 数据分析页 | 折线图表 | #6366f1 折线、#4f46e5 数据点、悬停标签 |
| 观众购票页 | 电子票卡片 | 320×180px、渐变色背景、隐形水印 |

### 4.3 响应式设计

- 桌面端优先设计
- 768px 以下转为单列堆叠布局
- 所有卡片和文本自适应缩放
- 移动端触控优化

### 4.4 性能要求

- 首页渲染时间不超过 1.5 秒
- 地图标记拖动时帧率不低于 45fps
- 数据加载使用骨架屏或加载动画
