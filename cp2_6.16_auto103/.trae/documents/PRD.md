# 多波段星图对比应用 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品背景
天文爱好者在研究恒星时，常常需要对比同一天体在不同电磁波波段下的观测差异。现有的星图应用往往只展示单一波段（通常为可见光），难以直观地呈现恒星在红外、紫外、X射线等波段下的形态变化。本应用旨在解决这一痛点，提供多波段星图的并排对比与混合查看功能。

### 1.2 产品定位
面向天文爱好者和业余天文学家的多波段星图可视化工具，支持同一星空区域在可见光、红外、紫外、X射线四个波段下的对比观测，以及波段混合比例的实时调节。

### 1.3 目标用户
- 天文爱好者
- 学生和科普教育工作者
- 业余天文学家

## 2. 功能需求

### 2.1 星图区域选择
| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| FR-01 | 用户可通过输入赤经（RA，格式 HH:MM:SS）和赤纬（Dec，格式 DD:MM:SS）选择观测星空区域 | P0 |
| FR-02 | 输入为空时默认显示猎户座区域（RA=5.5h, Dec=0°） | P0 |
| FR-03 | 输入框旁提供示例坐标按钮，点击即填充猎户座参数 | P1 |
| FR-04 | 系统查询该区域（视场 5°×5°）内的所有恒星数据 | P0 |

### 2.2 多波段并排显示
| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| FR-05 | 页面默认以 2×2 网格展示四张 Canvas 星图 | P0 |
| FR-06 | 四张星图分别对应：可见光、红外、紫外、X射线波段 | P0 |
| FR-07 | 每张星图上方标签显示波段名称和当前波段下的恒星数量 | P0 |
| FR-08 | 点击任一星图可将其放大至全屏单图模式 | P1 |
| FR-09 | 全屏模式下点击返回按钮可恢复网格视图 | P1 |

### 2.3 波段混合模式
| 需求编号 | 功能描述 | 优先级 |
|---------|---------|--------|
| FR-10 | 控制面板提供混合比例滑块（范围 0-100%，步长 1%） | P0 |

| FR-12 | 混合星图展示在网格正中央，标注"混合模式" | P0 |
| FR-13 | 滑块变动触发重新绘制，响应时间 < 100ms | P0 |

### 2.4 恒星信息悬浮提示
| 需求编号 | 功能描述 | 优先级 |

|---------|---------|--------|

| FR-20 | 控制面板下方以标签形式展示历史记录 | P0 |

## 3. 界面设计需求

### 3.1 整体主题
- **设计风格**：深色太空主题
- **主背景色**：#0f172a
- **星空粒子动画**：200 个白色小点，尺寸 1-2px 随机，透明度 0.3-0.8 随机，移动速度 0.1px/s

### 3.2 顶部导航栏
- 固定高度 60px
- 背景 #1e293b
- 底部 1px 分隔线 #334155
- 左侧：应用名称（白色 16px，字重 600）
- 右侧：当前观测区域坐标

### 3.3 星图区域
- 占页面主体部分
- Canvas 尺寸：最小 400×400px，响应式缩放，最小宽度 320px
- 四张星图间距 16px
- CSS Grid 布局（2 列等宽）
- 星图背景：#0f172a
- 恒星颜色：
  - 可见光：#facc15（金黄）
  - 红外：#ef4444（红）
  - 紫外：#8b5cf6（紫）
  - X射线：#06b6d4（青）
- 恒星大小：根据亮度缩放，最小 3px，最大 12px

### 3.4 控制面板
- 固定于页面右侧
- 宽度 280px
- 背景 #1e293b，圆角 16px，内边距 24px
- 滑块样式：
  - 轨道：高 6px，圆角 3px，背景 #334155
  - 滑块：直径 20px，背景 #3b82f6，1px 白色边框

### 3.5 交互反馈
- 所有按钮、滑块、输入框在点击/悬停时有平滑过渡（transition all 0.2s）

### 3.6 响应式布局
- 视口宽度 < 768px 时：
  - 四张星图改为单列排列
  - 控制面板折叠为底部可展开抽屉
  - 抽屉高度 200px，展开动画 0.3s ease-out

## 5. 技术约束

- 前端框架：React + TypeScript
- 构建工具：Vite
- 后端：Express
- 数据存储：JSON 文件
- 前后端交互：REST API + 自定义 Hooks
- 端口：前端 5173

## 6. 非功能需求

- 所有代码使用 TypeScript 编写，启用严格模式
- 文件结构按职责模块化
- 各文件间调用关系和数据流向需在代码中清晰体现
- 深色主题下需保证文字可读性（对比度 ≥ 4.5:1）
