## 1. 产品概述

在线小说章节自动排版与多格式导出应用，为作者和读者提供便捷的小说文本排版工具。用户粘贴纯文本小说后，系统自动识别章节段落、处理格式，并支持一键导出为适合电子书阅读的epub格式和适合印刷的PDF格式。

- 解决问题：纯文本小说格式混乱、阅读体验差、导出格式单一
- 目标用户：网络小说作者、文学爱好者、自出版创作者
- 产品价值：提升文本排版效率，提供专业级排版效果，支持多格式导出

## 2. 核心功能

### 2.1 功能模块

1. **编辑器模块**：文本编辑区，支持粘贴纯文本，自动段落格式化，字数统计
2. **预览模块**：实时排版预览，模拟纸书阅读效果，支持样式切换动画
3. **模板选择模块**：预设排版模板（经典文学、现代畅销），支持自定义参数调整
4. **导出模块**：一键导出epub和PDF格式，显示导出进度

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 主应用页面 | 编辑器 | contenteditable div实现，自动检测段落缩进和章节分隔，debounce处理文本变化 |
| 主应用页面 | 预览区 | iframe显示CSS样式效果，接收格式化文本，0.4秒淡入切换样式 |
| 主应用页面 | 模板选择器 | 从后端获取模板列表，选择后实时应用到预览 |
| 主应用页面 | 参数调整 | 段落间距滑块、正文字号步进按钮、页边距下拉选择 |
| 主应用页面 | 导出按钮 | 调用后端导出API，返回文件流下载，显示进度条 |

## 3. 核心流程

用户粘贴纯文本 → 系统自动识别段落和章节 → 选择排版模板 → 调整自定义参数 → 实时预览效果 → 点击导出 → 后端生成文件 → 浏览器下载。

```mermaid
flowchart LR
    A["用户粘贴文本"] --> B["自动识别段落/章节"]
    B --> C["选择排版模板"]
    C --> D["调整参数（间距/字号/边距）"]
    D --> E["实时预览"]
    E --> F["点击导出"]
    F --> G["后端生成epub/PDF"]
    G --> H["浏览器下载文件"]
```

## 4. 用户界面设计

### 4.1 设计风格

- 主背景色：#faf3e0（暖米色）
- 卡片背景色：#fff8e1（淡黄色）
- 编辑器背景：#f8f8f8（浅灰色）
- 字体：正文使用宋体/思源宋体，标题使用黑体
- 按钮：圆角8px，悬停亮度提升15%，点击缩放效果transform: scale(0.98) 0.1s
- 整体风格：温馨书卷气息，模拟传统阅读体验

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 主应用页面 | 整体布局 | 左右两栏，左60%右40%，响应式切换上下排列 |
| 主应用页面 | 编辑器 | 宽80%居中，圆角8px，背景#f8f8f8，内边距20px，右下角字数统计（灰色12px） |
| 主应用页面 | 预览区 | 等比例缩放页面，模拟纸书效果，淡入动画0.4s |
| 主应用页面 | 模板选择 | 两个预设模板卡片，选中状态高亮 |
| 主应用页面 | 参数调整 | 滑块、数字步进按钮、下拉选择，平滑过渡动画0.3s ease |
| 主应用页面 | 导出按钮 | 进度条（<50%橙色，>50%绿色） |

### 4.3 响应式设计

- Desktop-first设计
- 屏幕宽度 < 1024px时，Editor和Preview上下排列，各占100%宽度
- 触摸优化：滑块和按钮足够大，便于移动端操作

## 5. 性能要求

- 编辑器输入延迟 ≤ 50ms（使用300ms debounce）
- 预览样式切换动画 ≤ 0.5秒完成
- 导出操作 ≤ 3秒内返回文件
