## 1. 产品概述

三维地形生成与交互探索网页应用，解决传统地图展示缺乏立体感和沉浸感的问题，让用户能够直观理解地形起伏与等高线分布。目标用户为地理信息系统分析师、教育工作者和三维可视化爱好者。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 浏览地形、交互探查、等高线叠加 |

### 2.2 功能模块

1. **三维地形场景页**：地形加载渲染、视角旋转缩放平移、高度探查、等高线叠加
2. **信息面板**：相机坐标、海拔范围、顶点数、重置视角按钮

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 三维地形场景 | 地形加载与渲染 | 从CSV加载1024x1024高度数据，生成带纹理与光照的地形网格 |
| 三维地形场景 | 视角控制 | 鼠标拖拽旋转、滚轮缩放、右键平移，平滑跟随 |
| 三维地形场景 | 高度探查工具 | 鼠标悬停显示三维坐标和海拔，半透明圆环标记 |
| 三维地形场景 | 等高线叠加工具 | 按指定间隔生成发光等高线，颜色蓝到红渐变 |
| 信息面板 | 数据监控 | 实时显示相机坐标、海拔范围、顶点数 |
| 信息面板 | 视角重置 | 一键重置到鸟瞰位置，1.5秒缓动动画 |
| 系统功能 | 性能自适应 | 顶点超100万自动LOD切换，边缘显示状态提示 |

## 3. 核心流程

用户打开页面 → 加载CSV高度数据 → 生成地形网格与纹理 → 渲染三维场景 → 用户通过鼠标交互探索地形 → 实时查看高度信息 → 开启等高线叠加 → 系统根据性能自动调整LOD

    "保持最高LOD" --> "交互探索"

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深色科技风（背景 #0d0d1a，面板 #1a1a2e，文字 #c8c8e0）
- 按钮风格：渐变色按钮（#667eea 到 #764ba2），悬停上浮动效
- 字体：JetBrains Mono（数据展示）+ Source Sans 3（UI文字），大小12-16px
- 布局风格：左侧全屏三维场景 + 右侧半透明毛玻璃信息面板（280px宽）
- 地形纹理：低海拔绿色 → 高海拔白色渐变
- 等高线：蓝色外发光线条，颜色从低到高蓝→红渐变
- 探查标记：淡黄色半透明圆环

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 三维地形场景 | 地形网格 | Three.js Canvas全屏，高度渐变纹理，平行光+环境光 |
| 三维地形场景 | 高度探查 | 淡黄色半透明圆环标记，浮动信息提示框 |
| 三维地形场景 | 等高线 | 发光线条叠加在地形表面，蓝红渐变色 |
| 信息面板 | 数据区域 | 毛玻璃卡片，JetBrains Mono字体，实时数值 |
| 信息面板 | 重置按钮 | 渐变色圆角按钮，hover上浮效果 |
| 三维地形场景 | LOD提示 | 地形边缘文字，0.3秒淡入过渡 |

### 4.3 响应式设计

- 桌面优先设计，三维场景自适应剩余空间
- 信息面板在小屏时可通过折叠隐藏
- 鼠标交互为主，触控为辅

### 4.4 三维场景指导

- 环境：深色背景 #0d0d1a，雾效增加深度感
- 光照：平行光（方向从左上方45°）+ 环境光（柔和补光）
- 相机：透视相机，默认鸟瞰45°俯角，FOV 60°
- 交互：OrbitControls旋转缩放平移，Raycaster鼠标拾取
- 后处理：等高线发光效果，地形纹理淡入过渡
- 性能预算：45FPS以上，LOD切换150ms内
