## 1. 架构设计

纯前端Canvas应用，采用分层架构：

```mermaid
graph TD
    "UI层 (ui.ts)" --> "主控层 (main.ts)"
    "主控层 (main.ts)" --> "水墨渲染层 (ink.ts)"
    "主控层 (main.ts)" --> "流光粒子层 (flow.ts)"
    "水墨渲染层 (ink.ts)" --> "Canvas API"
    "流光粒子层 (flow.ts)" --> "Canvas API"
    "主控层 (main.ts)" --> "历史记录栈"
    "主控层 (main.ts)" --> "导出模块"
```

## 2. 技术栈说明

- **构建工具**：Vite 5+（轻量快速，原生ES模块支持）
- **开发语言**：TypeScript 5+（严格模式 `strict: true`）
- **渲染引擎**：原生 HTML5 Canvas 2D API（无第三方依赖）
- **状态管理**：内存中的历史栈数据结构（无框架）
- **性能优化**：
  - 离线Canvas（OffscreenCanvas）用于晕染扩散预计算
  - requestAnimationFrame 统一渲染循环
  - 粒子池复用，数量上限1500

## 4. 核心数据结构

### 4.1 水墨笔触
```typescript
interface InkPoint {
  x: number;
  y: number;
  pressure: number;      // 压力/速度系数
  timestamp: number;     // 生成时间戳
  brushSize: number;     // 笔刷大小
  color: string;         // 主颜色 rgba
}

interface InkStroke {
  id: string;
  points: InkPoint[];
  color: string;
  brushSize: number;
  createdAt: number;
  bloomProgress: number; // 晕染进度 0~1
  opacity: number;       // 当前不透明度（用于撤销淡出）
}
```

### 4.2 流光粒子
```typescript
interface FlowParticle {
  x: number;
  y: number;
  vx: number;            // x方向速度
  vy: number;            // y方向速度
  size: number;          // 粒子大小 2~5
  color: string;         // 当前颜色
  startColor: string;    // 起始颜色（笔刷色）
  endColor: string;      // 结束颜色（互补色）
  progress: number;      // 颜色渐变进度 0~1
  speed: number;         // 流动速度
  life: number;          // 剩余生命周期
  maxLife: number;       // 最大生命周期
  opacity: number;       // 当前不透明度
  trail: Array<{x: number, y: number, opacity: number}>; // 拖尾点
}
```

### 4.3 历史记录
```typescript
interface HistorySnapshot {
  strokes: InkStroke[];
  particles: FlowParticle[];
}

class HistoryManager {
  private undoStack: HistorySnapshot[];   // 撤销栈
  private redoStack: HistorySnapshot[];   // 重做栈
  private readonly MAX_STEPS = 20;        // 最多记录20步
}
```

## 5. 性能优化策略

### 5.1 晕染扩散优化
- 使用预渲染的径向渐变图案作为扩散遮罩
- 离屏Canvas预先渲染不同大小/透明度的晕染图案池
- 扩散计算使用时间插值，每帧只做简单绘制而非逐像素计算

### 5.2 粒子系统优化
- 粒子对象池复用（避免频繁GC）
- 活跃粒子数 > 1500 时，淘汰生命周期最短的旧粒子
- 拖尾使用链表存储，限制最大20节点
- 颜色渐变预计算LUT（查找表）

### 5.3 渲染循环
- 单 `requestAnimationFrame` 驱动全部渲染
- 双缓冲策略：先渲染到离屏Canvas，再整体拷贝到显示Canvas
- 脏区域标记（可选）：仅重绘变化区域
