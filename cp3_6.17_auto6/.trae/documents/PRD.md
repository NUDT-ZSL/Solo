# ClipMarker 产品需求文档（PRD）

## 1. 产品概述

ClipMarker 是一款面向音视频创作者的本地化视频素材标记与剪辑草稿工具，旨在解决大量原始素材难以检索、人工整理耗时的问题。用户可上传视频、在时间轴上添加彩色标签标记，并一键导出 JSON 格式的剪辑时间线草稿，供后续在专业剪辑软件中导入使用。

- 目标用户：音视频创作者、剪辑师、内容工作室
- 核心价值：将"看素材→打标签→整理片段"的人工流程从数小时缩短至分钟级，提升素材检索效率与剪辑前期准备效率
- 市场定位：轻量级本地工具，零云端依赖，隐私友好，快速上手

## 2. 核心功能

### 2.1 功能模块

1. **视频上传与管理**：支持拖拽/点击上传 MP4/MOV，单文件 ≤200MB，横向卡片展示视频信息
2. **视频播放与标记**：模态播放器，支持快捷键 M 添加时间戳标签，预设 10 种彩色标签
3. **标记侧边栏管理**：按视频分组、按时间排序，支持跳转、拖拽排序、删除
4. **时间线导出**：多选标记片段，导出 JSON 剪辑草稿（含帧级时间、标签、排序）

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主工作区 | 视频上传区 | 拖拽/点击上传，文件格式与大小校验，上传进度 |
| 主工作区 | 视频卡片列表 | 横向卡片展示（320×180px），含缩略图、文件名、时长、大小、播放按钮 |
| 主工作区 | 模态播放器 | 640×360px 播放器，进度条带彩色标记竖线，添加标记按钮与标签弹框 |
| 侧边栏 | 标记列表 | 240px 宽，按视频分组按时间排序，标记行含时间戳、标签、缩略图，支持跳转/拖拽/删除 |
| 顶部工具栏 | 导出控制 | 多选标记后导出 timeline.json |

## 3. 核心流程

用户打开应用 → 上传视频文件 → 视频以卡片展示 → 点击播放按钮打开模态播放器 → 播放时按 M 键或点击"添加标记" → 标签弹框选择/输入标签 → 标记以彩色竖线显示在进度条 → 标记汇总到右侧边栏 → 在侧边栏多选标记 → 点击"导出时间线"生成 timeline.json 下载。

## 4. 用户界面设计

### 4.1 设计风格

- **主题**：暗色专业工作站风格（DaVinci Resolve / Premiere Pro 工作流灵感）
- **主色**：背景 `#121212`，卡片 `#1e1e1e`，边栏 `#252525`，主文字 `#e0e0e0`，次文字 `#9e9e9e`
- **强调色**：`#ff5722`（播放按钮、主操作按钮、激活态）
- **标签色板**：10 色渐变 `#e53935` → `#1e88e5`（红→蓝渐变）
- **按钮交互**：点击时 `transform: scale(0.95)`，`transition: 0.2s`
- **字体**：系统等宽与无衬线混合，数字时间戳使用等宽字体
- **圆角**：卡片 8px，标签 12px，按钮 50%
- **布局**：左右结构，左侧 75%（上传区+播放器+卡片列表），右侧固定 240px 边栏

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|
| 主工作区 | 上传区 | 虚线边框拖拽区，深灰背景，悬停高亮 #2a2a2a，支持点击选择 |
| 主工作区 | 视频卡片 | 320×180px，圆角 8px，#1e1e1e 背景，左下圆形播放按钮（36px，#ff5722），右侧文件信息 |
| 模态播放器 | 播放器 | 640×360px，黑色背景，底部进度条带彩色标记竖线（3px），时间戳显示 |
| 标签弹框 | 标签选择 | 输入框 + 10 个预设标签（60×24px，圆角 12px，渐变色） |
| 侧边栏 | 标记列表 | 240px，#252525，内边距 12px，按视频分组，标记行含 32×32 缩略图 |
| 顶栏 | 导出按钮 | #ff5722 强调，禁用态灰色，点击 scale 0.95 |

### 4.3 响应式

- 桌面优先（≥768px）：左右双列布局，左侧 75% 自适应，右侧固定 240px
- 移动端（<768px）：自动切换为单列上下滚动，边栏移至底部
- 触摸优化：拖拽区域增大点击热区，按钮最小 36px 触摸目标

### 4.4 性能要求

- 视频播放与标记拖拽操作刷新率 ≥30FPS
- 使用 requestAnimationFrame 优化进度条更新
- 缩略图懒加载，避免一次性渲染大量图片

## 5. 技术约束

- 前端：TypeScript + React + Vite，端口 3000
- 后端：Express + multer 文件上传 + JSON 文件存储，端口 4000
- 前后端通过 `/api` 代理通信
- 文件组织严格遵循用户指定的结构
- 启动方式：`npm install && npm run dev`
