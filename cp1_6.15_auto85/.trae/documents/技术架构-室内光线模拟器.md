## 2. 技术描述

- **前端框架**：原生 TypeScript (无UI框架)，保持轻量高效
- **3D渲染引擎**：Three.js r160+，使用WebGLRenderer + PBR材质体系
- **构建工具**：Vite 5.x，热更新HMR，TypeScript编译
- **状态管理**：无第三方库，采用模块化内部状态 + 观察者回调模式
- **动画系统**：自研 requestAnimationFrame 驱动的缓动动画管理器

**技术选型理由**：
- 纯Three.js而非React Three Fiber：用户明确指定独立业务逻辑模块，渲染与业务解耦更清晰
- 自研动画系统：Three.js内置动画系统偏重关键帧，自研缓动更适合实时参数过渡
- 原生UI组件：避免引入UI库导致的体积膨胀，满足2秒首屏加载要求

## 4. 模块间数据接口定义

### 4.1 LightController 接口

```typescript
// 光源参数
interface LightParams {
    x: number;      // X轴位置 (-6 ~ 6)
    y: number;      // Y轴位置 (1 ~ 7)
    z: number;      // Z轴位置 (-5 ~ 5)
    intensity: number;  // 强度 (0 ~ 5)
}

// 边界检测结果
interface BoundaryResult {
    params: LightParams;       // 校正后参数
    violated: {                // 被触发的边界
        xMin: boolean; xMax: boolean;
        yMin: boolean; yMax: boolean;
        zMin: boolean; zMax: boolean;
    };
}

// 房间尺寸常量
const ROOM = { width: 12, height: 8, depth: 10 }
```

## 5. 文件结构与调用关系

```
project/
├── package.json              # 依赖配置: three, typescript, vite, @types/three
├── vite.config.js            # Vite基础配置
├── tsconfig.json             # strict模式TS配置
├── index.html                # 入口页面，全屏canvas容器
└── src/
    ├── main.ts               # [渲染入口] → 初始化场景/相机/渲染器 → 调用sceneSetup
    │                           → 订阅animationManager更新 → 每帧render
    ├── sceneSetup.ts         # [3D场景] → 创建地板/墙壁/立方体/球体/光源/阴影
    │                           → 暴露updateLight(params)方法供animationManager调用
    ├── lightController.ts    # [业务逻辑] → 接收UI输入 → 边界约束 → 输出校正参数
    │                           → 方案存储/读取 → 触发animationManager过渡
    ├── animationManager.ts   # [动画逻辑] → 独立于Three.js的缓动系统
    │                           → 管理光源位置/强度/阴影柔化多个动画时间线
    └── interactionPanel.ts   # [UI层] → 创建DOM面板 → 滑动条/按钮事件监听
                                → 调用lightController → 显示边界震动反馈
```

**数据流向**：`UI交互` → `interactionPanel.ts` → `lightController.ts`(边界校验) → `animationManager.ts`(缓动插值) → `sceneSetup.ts`(更新Three.js对象) → `main.ts`(WebGL渲染)

## 6. 性能优化策略

### 6.1 渲染性能
- ShadowMap尺寸：2048×2048，平衡质量与性能
- 阴影相机：OrthographicShadowCamera，近远平面裁剪
- 材质复用：相同PBR参数的几何体共享Material实例
- MatrixAutoUpdate：静态物体关闭自动矩阵更新

### 6.2 加载性能
- Three.js按需import：仅引入必要模块(Object, Material, Light, Control)
- 无第三方UI库：原生DOM + CSS实现控制面板
- 首屏关键CSS内联到index.html `<style>` 标签

### 6.3 响应性能
- 滑动条节流：requestAnimationFrame合并高频输入事件
- 动画复用：相同目标值不重复创建动画任务
- 边界检测O(1)：简单数值比较，无复杂计算
